a)

Objekte der Klasse Zug werden in einem Attribute der Klasse Verwaltung in einer Liste List<Zug> gespeichert.
Objekte der Klasse Verwaltung greift auf die Züge über diese Liste zu.

Objekte der Klasse Waggon werden in einem Attribute der Klasse Zug in einer Liste List<Waggon> gespeichert.
Objekte der Klasse Zug greift auf die Züge über diese Liste zu.

Die Klasse Waggon hat einen array von Objekten der Klasse Sitzplatz.

Die List hat zum Array den bedeutenden Unterschied, dass diese skalierbar ist.
So kann die Verwaltung dank der Liste Züge problemlos und effizient hinzufügen und löschen.

Bei den Sitzplätzen in einem Waggon ist dies nicht nötig, da die Anzahl der Sitzplätze in einem Waggon sich für gewöhnlich nicht ändert.

c)

Ansatz:

Eine Methode, die nach freien Vierergruppen sucht sollte durch die Waggons iterieren und für die Plätze immer 4 Plätze am Stück überprüfen. Das letztere kann mithilfe einer Schleife getan werden.
Deutlich effizienter jedoch wäre eine Überprüfung aller 4 Plätze gleichzeitig mithilfe einer Bedingung.

Code:

content_copy
public int[] ermittleFreieVierergruppe() {
  for (waggons.toFirst(); waggons.hasAccess(); waggons.next()) {
    Waggon current = waggons.getContent();
    for (int i = 0; i < current.gibAnzahlPlaetze(); i += 4) {
      if (current.istPlatzFrei(i)
        && current.istPlatzFrei(i + 1)
        && current.istPlatzFrei(i + 2)
        && current.istPlatzFrei(i + 3)
      ) return new int[]{current.gibNummer(), i};
    }
  }
  return new int[]{-1, -1};
}

Da Kunden nun eindeutig mithilfe der Kundennummer repräsentiert werden, sollten auch die Sitzplätze mit den Kundennamen assoziiert werden.
Dies heißt, dass die Reserviertungsmethoden der Klasse Zug und der Klasse Waggon verändert werden müssen, sodass sie Objekte der Klass Kunde als Parameter nehmen und die Kundennummer so entnehmen.

Die Klasse Sitzplatz muss dann ebenfalls so verändert werden, dass sie entweder mit der Kundennummer arbeitet, sodass reservirtFuer vom Typ int ist und die Kundennummer repräsentiert.
Dementsprächend müssen die Methoden trageReservierungEin(...) und gibReservierungFuer() auch im Diagram verändert werden.

e)

Die Modellierung von Thomas scheint mir eher unübersichtlich, da es sehr vieles erschwert und nichts vereinfacht. Um zum Beispiel auf alle Sitzplätze eines Waggons zuzugreifen, müssten wir durch alle Sitzplätze iterieren, und diese ausfiltern. Das ist viel ineffizienter, als auf einen Waggon zuzugreifen, und dort die Sitzplätze zu nutzen.

Die Modellierung von Thomas würde in einer Datenbank vielleicht mehr Sinn ergeben, aber nicht in einer Objekt-Orientierten Sprache.
